# CAJAL Figure Report — Appendix: Best Practices for Agentic Book Repos
_Density: recommend 1 figure, Foundational._

This appendix is an operating compact: two-customers framing, verified-data-first and vetted-scripts-first ordering rules, a seven-gate phase sequence, and logging guidance. Most of it is declarative policy. The one genuinely figure-worthy concept is the seven phase gates (MC — an ordered, interdependent sequence that must pass before a fully automated pipeline runs). The two-customers idea is repeated from Chapter 4 / Chapter 97 and does not warrant a third figure here.

## Figure 1 — The Seven Phase Gates
Priority: Important · Type: process flowchart

BLOCK 1 — ILLUSTRAE PASTE BLOCK:
Render a left-to-right (or top-to-bottom) sequential gate flowchart of the seven phase gates that must pass before a fully automated pipeline runs. Draw seven gate nodes in order: problem gate, local evidence gate, stored script gate, small-run gate, verification gate, review gate, and logging gate. Connect them with single-headed arrows showing strict sequence, and show that the pipeline-automation terminal is reachable only after the seventh gate. At each gate, draw a short branch indicating a failed gate halts progression and does not advance to the next gate. Make the gate sequence the spine of the figure, with the halt branches as secondary blocking cues. Keep exactly seven gates plus the terminal, evenly spaced and clearly ordered at 89mm. Use color to mark the gate spine, the pass path, and the halt branches. Flat vector, colorblind-safe palette, no baked text, white background, 1pt strokes.

BLOCK 2 — FULL SCOPE:
[S - SPECIFICATION] default textbook: single-column 89mm, ≥300 DPI, vector, Okabe-Ito.
[C - CONTENT] seven ordered gates: problem gate, local evidence gate, stored script gate, small-run gate, verification gate, review gate, logging gate; automated-pipeline terminal reachable only after the seventh gate; each gate has a halt branch on failure.
[O - ORGANIZATION] sequential spine of seven gate nodes → pipeline terminal; each gate has a blocking halt branch; → progression, ⊣ at halts.
[P - PRESENTATION] flat vector; gate spine dominant #0072B2; pass path active #009E73; halt branches blocking #D55E00; terminal primary #56B4E9; 1pt strokes; white background; NO baked text.
[E - EXCLUSIONS] omit the two-customers framing; omit verified-data-first / vetted-scripts-first lists; omit logging-detail content; omit dollar figures; omit any repo file paths as text.

BLOCK 3 — NEGATIVE PROMPT:
gate/fence photos, file-path text, dollar amounts, text labels, words, gibberish letters, titles, captions, decorative borders, realistic textures, drop shadows, gradient backgrounds, photographic elements, dual-headed arrows, hand-drawn styles, human figures, visual clutter, watermarks, red-green color combinations, rainbow color scales, 3D perspective distortion.
